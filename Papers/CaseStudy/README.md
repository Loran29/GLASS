# Case Study Corpus

This folder contains 24 academic papers used as the retrieval-augmented generation (RAG) evidence corpus for the second stage of the pipeline. Each paper documents a real business process simulation case study — quantitative results from these papers ground the scenario proposals generated by the second LLM.

## Purpose

The papers are indexed by [`goal_to_parameters/knowledge/pdf_indexer.py`](../../goal_to_parameters/knowledge/pdf_indexer.py) and retrieved by the hybrid RAG system ([`goal_to_parameters/knowledge/hybrid_retrieval.py`](../../goal_to_parameters/knowledge/hybrid_retrieval.py)) — BM25 + dense embeddings + cross-encoder reranking over paragraph-level chunks. Structured metadata for each paper (title, authors, year, domain, quantitative evidence) lives in [`goal_to_parameters/knowledge/kb_data.py`](../../goal_to_parameters/knowledge/kb_data.py).

## Behavior when the corpus is absent

The RAG code fails gracefully — if this folder is empty or removed, `load_pdf_chunks` returns an empty list and retrieval falls back to the structured knowledge base entries in `kb_data.py`. The pipeline still runs, but full-text chunk retrieval is disabled and grounding is weaker.
